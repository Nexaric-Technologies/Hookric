# Product Requirements Document (PRD)

## Product Name

Hookrick — by nexaric

## Vision

Allow developers, agencies, automation builders, n8n users, Zapier users, WooCommerce users, WhatsApp API users, and SaaS developers to instantly create a unique webhook endpoint and inspect every incoming request in real time without requiring registration or database storage.

The product should work immediately after opening the website.

# Request Capture Requirements

For every request capture maximum possible information.

## Basic Information

* Unique Request ID
* Timestamp
* Processing Time
* Request Method
* URL
* Original URL
* Path
* Host
* Protocol
* HTTP Version

Example:

Method: POST
URL: /r/abc123
Protocol: HTTPS
HTTP Version: 1.1

---

## Network Information

Capture:

* Source IP
* Forwarded IP
* X-Forwarded-For
* Remote Address
* Country
* Region
* City
* ASN
* ISP

Example:

Country: India
State: West Bengal
City: Kolkata

---

## TLS Information

Capture if available:

* SSL Version
* Cipher
* TLS Version
* Secure Connection

---

## Header Information

Store ALL headers.

Example:

Authorization
User-Agent
Content-Type
X-Hub-Signature
X-Shopify-Topic
X-WC-Webhook-ID

Display searchable JSON.

---

## Query Parameters

Capture:

GET /webhook?id=100&status=paid

Display:

{
"id":"100",
"status":"paid"
}

---

## Cookies

Capture all cookies.

Example:

session_id
auth_token
tracking_id

---

## Raw Request Body

Store exact request body.

Example:

Raw Text

Raw JSON

Raw XML

Raw Form Data

Raw Multipart

User can toggle between:

* Raw
* Pretty JSON
* Parsed

---

## JSON Analysis

If body is JSON:

* Total keys

* Arrays
